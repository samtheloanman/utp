<objective>
Implement the `CollateralManager.sol` smart contract responsible for handling collateral and minting `UBTC` tokens. This plan also includes creating a deployment script to deploy both `UBTC.sol` and `CollateralManager.sol` and correctly set their ownership.

Purpose: To create the core logic that backs the UBTC stablecoin with collateral and to provide a repeatable way to deploy the system.
Output: A deployable `CollateralManager.sol` contract, corresponding unit tests, and a master deployment script.
</objective>

<task type="auto" tdd="true">
  <name>Task 1: Implement CollateralManager.sol</name>
  <files>
    - "contracts/CollateralManager.sol"
  </files>
  <behavior>
    - It must store the address of the `UBTC` token contract.
    - It must have a `payable` function, `deposit()`, that accepts BTC (from the RSK bridge, simulated with native testnet tokens for now).
    - When `deposit()` is called, it should mint an equivalent amount of `UBTC` tokens to the caller's address. (Assuming 1:1 for now).
    - It should only be able to mint tokens if it is the owner of the `UBTC` contract.
  </behavior>
  <action>
  1. Create the `contracts/CollateralManager.sol` file.
  2. Define an interface `IUBTC` with the `mint(address, uint256)` function signature.
  3. The `CollateralManager` constructor will accept the address of the `UBTC` contract.
  4. Implement the `deposit()` payable function. Inside this function, it will call `IUBTC(ubtcAddress).mint(msg.sender, msg.value)`.
  5. For now, the integration with the RSK bridge is simplified: we treat `msg.value` (the native gas token on RSK testnet, tRBTC) as the collateral being deposited.
  </action>
  <verify>
    <automated>npx hardhat compile</automated>
  </verify>
  <done>
    The `CollateralManager.sol` contract is created and compiles successfully.
  </done>
</task>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EOA -> `CollateralManager.sol` | User sends collateral to the manager contract. |
| `CollateralManager.sol` -> `UBTC.sol` | Manager contract calls `mint` on the token contract. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Tampering | `CollateralManager.sol::deposit` | Mitigate | The amount of UBTC minted is directly tied to `msg.value`, preventing the contract from minting arbitrary amounts. Re-entrancy is not a major risk here as the external call to `mint` is the last step, but for best practice, we follow a checks-effects-interactions pattern. |
| T-01-06 | Information Disclosure | Contract State | Accept | The addresses of the UBTC token and the total collateral are public on the blockchain, which is intended for transparency. |
</threat_model>
