<objective>
Create the core `UBTC.sol` token contract, which will serve as the project's stablecoin. This contract will be a standard ERC20 token with added controls for minting and burning, restricted to an owner address (which will later be a controller contract).

Purpose: To establish the fundamental token asset for the stablecoin ecosystem.
Output: A deployable and testable `UBTC.sol` smart contract and corresponding unit tests.
</objective>

<task type="auto" tdd="true">
  <name>Task 1: Implement UBTC.sol ERC20 Contract</name>
  <files>
    - "contracts/UBTC.sol"
  </files>
  <behavior>
    - It MUST be an ERC20 token named "United Bitcoin" with the symbol "UBTC".
    - It MUST inherit from OpenZeppelin's `ERC20.sol` and `Ownable.sol`.
    - It MUST have a `mint(address to, uint256 amount)` function, callable only by the owner, to create new tokens.
    - It MUST have a `burn(address from, uint256 amount)` function, callable only by the owner, to destroy tokens.
    - The initial supply should be zero.
  </behavior>
  <action>
  1. Create the `contracts/UBTC.sol` file.
  2. Use the OpenZeppelin v5.x contracts library, which is already in the project context.
  3. Implement the `UBTC` contract inheriting from `ERC20` and `Ownable`.
  4. The constructor should initialize the `ERC20` with the name "United Bitcoin" and symbol "UBTC".
  5. Implement the `mint` function, guarded by the `onlyOwner` modifier, which calls the internal `_mint` function.
  6. Implement the `burn` function, guarded by the `onlyOwner` modifier, which calls the internal `_burn` function.
  </action>
  <verify>
    <automated>The file `contracts/UBTC.sol` should compile successfully using `npx hardhat compile`.</automated>
  </verify>
  <done>
    The `UBTC.sol` contract file is created, compiles without errors, and contains the required functions and modifiers.
  </done>
</task>

## STRIDE Threat Register

|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering / Elevation of Privilege | `UBTC.sol::mint` | Mitigate | The `mint` function is protected by OpenZeppelin's `Ownable` modifier (`onlyOwner`), ensuring only the designated controller can create new tokens. |
| T-01-02 | Tampering / Elevation of Privilege | `UBTC.sol::burn` | Mitigate | The `burn` function is also protected by the `onlyOwner` modifier, preventing unauthorized token destruction. |
| T-01-03 | Repudiation | `UBTC.sol` | Mitigate | All state-changing actions (mint, burn, transfer) emit ERC20 standard events (`Transfer`), providing an on-chain audit trail. |
</threat_model>
